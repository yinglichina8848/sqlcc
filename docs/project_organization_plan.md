# SQLCC项目目录整理计划

## 目标
全面分析并重新组织项目的目录结构，使其更加清晰合理。

## 任务清单

### 1. 现状分析
- [ ] 检查当前项目目录结构
- [ ] 检查Git状态，确认缺失的源码目录
- [ ] 从Gitee恢复丢失的include和src目录
- [ ] 分析现有文档和报告分布

### 2. 目录结构设计
- [ ] 设计新的目录结构
- [ ] 规划docs目录的子分类
- [ ] 规划include目录的模块化结构
- [ ] 规划src目录的模块化结构
- [ ] 规划tests目录的测试分类

### 3. 文档整理
- [ ] 将主目录下的文档移动到docs目录
- [ ] 按类别和版本组织文档
- [ ] 整理Shell脚本和SQL脚本

### 4. 源码整理
- [ ] 创建include目录的模块化子目录结构
- [ ] 重新组织src目录的源码文件
- [ ] 更新源码中的include路径
- [ ] 避免相对路径，使用环境变量

### 5. 测试整理
- [ ] 整理tests目录结构
- [ ] 按测试类型分类：单元测试、集成测试、性能测试、覆盖率测试
- [ ] 移动cpp和shell测试文件

### 6. 构建文件更新
- [ ] 更新CMakeLists.txt文件
- [ ] 调整源码文件路径配置
- [ ] 更新include目录配置

### 7. 验证和测试
- [ ] 验证编译是否正常
- [ ] 确保所有路径正确
- [ ] 检查测试是否正常运行

## 设计原则
- 简洁清晰的目录结构
- 模块化的组织方式
- 避免复杂的相对路径
- 便于维护和扩展
